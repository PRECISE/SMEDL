SMEDL Programming InterfaceÂ¶

These pages describe how to interface with the generated monitors. Most programs will only need the information from Essential API and the page for the chosen asynchronous transport. The Full API page is useful for those who want a better understanding of how SMEDL monitors work internally or who want to integrate with the generated code on a deeper level.

Note

Many file names and identifiers in this API contain placeholders for event, monitor, and synchronous set names. These placeholders are marked by the use of ALLCAPS.
